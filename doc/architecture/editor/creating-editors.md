# Creating New Card Editors

This guide covers creating visual editors for LCARdS cards using the editor boilerplate generator and manual methods.

## Quick Start with Generator

The fastest way to create a new editor is using the CLI boilerplate generator:

```bash
npm run create-editor
```

Or with a pre-specified card type:

```bash
npm run create-editor chart
```

### Interactive Prompts

The generator will guide you through configuration:

1. **Card Type**: Lowercase identifier (e.g., `chart`, `gauge`, `timeline`)
   - Must be unique (not already registered)
   - Lowercase, alphanumeric with hyphens only
   - Will create `lcards-{type}-editor.js`

2. **Display Label**: Human-readable name (e.g., "Chart Card")
   - Auto-generated from card type if not specified
   - Used in editor UI and documentation

3. **Has Primary Entity**: Whether the card controls/displays an entity
   - `Yes`: Adds entity selector field to schema
   - `No`: Omits entity field for static cards

4. **Tab Selection**: Choose tabs to include
   - ✓ **General**: Entity, name, basic config
   - ✓ **Style**: Appearance, colors, layout
   - ☐ **Data**: Data sources, series (for charts/graphs)
   - ✓ **Advanced**: Actions, templates

5. **Base Template**: Starting point
   - **Minimal**: Entity + name only, single tab
   - **Standard**: Entity + style basics + actions (recommended)
   - **Advanced**: Standard + data tab + templates
   - **Clone from slider/button/elbow**: Copy existing editor

6. **Generate Schema**: Create schema file with x-ui-hints
   - `Yes`: Creates `src/cards/schemas/{type}-schema.js`
   - `No`: Manual schema creation required

### Generated Files

The generator creates:

```
src/editor/cards/lcards-{type}-editor.js  ← Editor class
src/cards/schemas/{type}-schema.js         ← JSON Schema (if selected)
```

Example output:
```
✅ Created: src/editor/cards/lcards-chart-editor.js
✅ Created: src/cards/schemas/chart-schema.js

✨ Done! Next steps:
  1. Customize schema: src/cards/schemas/chart-schema.js
  2. Add tab content: src/editor/cards/lcards-chart-editor.js
  3. Test: Reload HA, create Chart Card, verify editor opens
  4. Use Schema Editor: open tools/schema-editor.html
```

## Generated Editor Structure

### Editor Class

```javascript
/**
 * Chart Card Editor
 * 
 * Visual editor for LCARdS Chart cards.
 * Generated by tools/create-editor.js on 2025-01-15
 * 
 * @extends LCARdSBaseEditor
 */

import { html } from 'lit';
import { LCARdSBaseEditor } from '../base/LCARdSBaseEditor.js';
import { LCARdSFormFieldHelper as FormField } from '../components/shared/lcards-form-field.js';
import '../components/shared/lcards-form-section.js';

export class LCARdSChartEditor extends LCARdSBaseEditor {

    constructor() {
        super();
        this.cardType = 'chart';
    }

    _getTabDefinitions() {
        return [
            { label: 'General', content: () => this._renderGeneralTab() },
            { label: 'Style', content: () => this._renderStyleTab() },
            { label: 'Advanced', content: () => this._renderAdvancedTab() },
            ...this._getUtilityTabs()
        ];
    }

    _renderGeneralTab() {
        return html`
            <lcards-form-section 
                header="Basic Configuration"
                description="Core card settings">
                
                ${FormField.renderField(this, 'entity', {
                    label: 'Entity',
                    helper: 'Select the primary entity for this chart'
                })}
                
                ${FormField.renderField(this, 'name', {
                    label: 'Name',
                    helper: 'Display name for the card (optional)'
                })}
                
                <!-- TODO: Add chart-specific fields here -->
                
            </lcards-form-section>
        `;
    }

    // Additional tab renderers...
}

customElements.define('lcards-chart-editor', LCARdSChartEditor);
```

## Customization

### Adding Fields

Use `FormField.renderField()` in tab renderers:

```javascript
_renderGeneralTab() {
    return html`
        <lcards-form-section header="Configuration">
            ${FormField.renderField(this, 'entity', {
                label: 'Entity',
                helper: 'Entity to display'
            })}
            
            ${FormField.renderField(this, 'chart_type', {
                label: 'Chart Type',
                helper: 'Select visualization type'
            })}
        </lcards-form-section>
    `;
}
```

### Adding Schema Properties

Edit the generated schema file:

```javascript
export const chartSchema = {
    type: 'object',
    properties: {
        // ... existing properties ...
        
        chart_type: {
            type: 'string',
            enum: ['line', 'bar', 'area', 'pie'],
            'x-ui-hints': {
                label: 'Chart Type',
                helper: 'Select visualization type',
                selector: {
                    select: {
                        options: [
                            { value: 'line', label: 'Line Chart' },
                            { value: 'bar', label: 'Bar Chart' },
                            { value: 'area', label: 'Area Chart' },
                            { value: 'pie', label: 'Pie Chart' }
                        ]
                    }
                }
            }
        }
    }
};
```

## Registration

### Self-Registration Pattern

LCARdS editors use self-registration. The editor must be imported by the card class:

**In `src/cards/lcards-chart.js`:**

```javascript
import { LCARdSCard } from '../base/LCARdSCard.js';

// Import editor for registration
import '../editor/cards/lcards-chart-editor.js';

export class LCARdSChart extends LCARdSCard {
    // Card implementation...
}

customElements.define('lcards-chart', LCARdSChart);
```

**Note**: The editor's `customElements.define()` at the bottom of the editor file handles registration.

## Templates

### Minimal Template

Best for: Simple cards with basic configuration

**Includes:**
- Entity field (optional)
- Name field
- Single General tab

**Use when:**
- Card has minimal configuration
- No complex styling options
- Testing/prototyping

### Standard Template (Recommended)

Best for: Most card types

**Includes:**
- Entity + name fields
- General, Style, Advanced tabs
- Background color, border radius, padding
- Tap/hold actions

**Use when:**
- Card has typical configuration needs
- Requires basic styling
- Standard interaction patterns

### Advanced Template

Best for: Complex cards with data/rules

**Includes:**
- All standard features
- Data tab for data sources
- Template field placeholders
- Rules integration hints

**Use when:**
- Card displays charts/graphs
- Uses data sources
- Requires template expressions

### Clone Mode

Best for: Similar cards to existing ones

**Process:**
1. Select source editor (slider, button, elbow)
2. Generator copies editor structure
3. Search/replace card type throughout
4. Preserves tab structure and field organization

**Use when:**
- Creating similar card variant
- Learning from existing patterns
- Need complex tab structure

## Manual Creation

If you prefer not to use the generator:

1. Create `src/editor/cards/lcards-{type}-editor.js`
2. Extend `LCARdSBaseEditor`
3. Implement `_getTabDefinitions()`
4. Create tab renderer methods
5. Add `customElements.define()` at bottom
6. Create schema in `src/cards/schemas/{type}-schema.js`
7. Import editor in card class

See existing editors for examples:
- **Simple**: `lcards-elbow-editor.js`
- **Medium**: `lcards-slider-editor.js`
- **Complex**: `lcards-button-editor.js`

## Architecture

### Editor Base Class

All editors extend `LCARdSBaseEditor` which provides:
- Tab management system
- Config state synchronization
- YAML coordination
- Schema validation
- FormField helpers
- Utility tabs (Developer, YAML, etc.)

### Tab System

Declarative tab definitions:

```javascript
_getTabDefinitions() {
    return [
        { label: 'Config', content: () => this._renderConfigTab() },
        { label: 'Style', content: () => this._renderStyleTab() },
        ...this._getUtilityTabs()  // Adds Developer, YAML, etc.
    ];
}
```

### FormField Rendering

Automatic field rendering based on config path:

```javascript
FormField.renderField(this, 'style.background.color', {
    label: 'Background Color',
    helper: 'Card background color'
})
```

The FormField helper:
- Reads schema for field configuration
- Renders appropriate selector (color, entity, text, etc.)
- Handles value updates
- Shows validation errors

### Schema Integration

Schemas define:
- Property types and constraints
- UI hints for rendering
- Selector configuration
- Validation rules

Example with `oneOf` for choices:

```javascript
border_radius: {
    oneOf: [
        {
            type: 'number',
            title: 'Pixels',
            minimum: 0,
            maximum: 50
        },
        {
            type: 'string',
            title: 'Theme Token',
            pattern: '^\\{theme:.*\\}$'
        }
    ],
    'x-ui-hints': {
        selector: {
            choose: {
                choices: {
                    pixels: { selector: { number: { /* ... */ } } },
                    theme: { selector: { text: { /* ... */ } } }
                }
            }
        }
    }
}
```

## Troubleshooting

### Editor Not Appearing

**Problem**: Card doesn't show in "Add Card" menu

**Solution**:
1. Verify editor imports in card class
2. Check `customElements.define()` is called
3. Rebuild: `npm run build`
4. Hard refresh Home Assistant

### Schema Validation Errors

**Problem**: Fields show validation errors

**Solution**:
1. Check schema property types match config
2. Verify required fields in schema
3. Use schema editor for visual validation
4. Check console for detailed errors

### Fields Not Rendering

**Problem**: FormField.renderField() shows nothing

**Solution**:
1. Verify schema exists for property path
2. Check x-ui-hints selector configuration
3. Ensure property path matches config structure
4. Check console for rendering errors

### Build Errors

**Problem**: `npm run build` fails

**Solution**:
1. Check JavaScript syntax in editor file
2. Verify all imports are correct
3. Ensure schema exports properly
4. Run `npm run build:dev` for detailed errors

## Best Practices

### Editor Organization

- **One concern per tab**: Don't mix unrelated settings
- **Logical grouping**: Group related fields in sections
- **Progressive disclosure**: Common settings first, advanced later
- **Consistent naming**: Follow existing card conventions

### Schema Design

- **Required fields**: Only mark truly required properties
- **Helpful descriptions**: Write clear helpers for each field
- **Sensible defaults**: Provide defaults for optional properties
- **Validation rules**: Use min/max, pattern, enum appropriately

### Field Helpers

- **Be specific**: "Background color for the card" vs "Color"
- **Mention format**: "Theme token like {theme:palette.moonlight}"
- **Show examples**: "e.g., #FF9900 or var(--lcars-orange)"
- **Explain impact**: "Higher values increase animation speed"

### Tab Structure

**General Tab**: Entity, name, core functionality
**Style Tab**: Colors, borders, spacing, fonts
**Data Tab**: Data sources, series, aggregations
**Advanced Tab**: Actions, templates, rules, animations
**Utility Tabs**: Auto-added (Developer, YAML, Rules, etc.)
